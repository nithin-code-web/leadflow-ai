Every feature implemented for this project should go through the following cycle:

1.  **Plan**
2.  **Implement**
3.  **Review**
4.  **Refactor**
5.  **Commit**
6.  **Push**

**Why:** This structured approach ensures quality, maintainability, and alignment with project goals. It promotes thoughtful development and reduces technical debt.

**How to apply:** I will follow these steps for every feature I work on, starting with the planning phase using `EnterPlanMode` for complex tasks, and ensuring explicit user approval before proceeding to implementation or committing changes.
